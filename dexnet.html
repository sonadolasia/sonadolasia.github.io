Dex-Net User Interface

January - April 2018

In Spring 2018, working as an undergratuate researcher in the UC Berkeley Automation Lab, I was given a unique project to designed and develop a graphical user interface for the Dexterity Network (Dex-Net). I worked in collaboration with Professor Ken Goldberg and graduate students; Jeff Mahler, Mike Danielczuk and Matt Matl.

PROJECT OVERVIEW

In 2018, the UC Berkeley AUTOLAB was invited to present the Dex-Net project at Amazon's invite-only MARS conference in Palm Springs. For the live demo the lab wanted a display that would sit on top of the robot, demonstrating the thought process of the project in real time. 

WHAT IS DEX-NET?

The Dexterity Network (Dex-Net) is a research project including code, datasets, and algorithms for generating datasets of synthetic point clouds, robot parallel-jaw grasps and metrics of grasp robustness based on physics for thousands of 3D object models to train machine learning-based methods to plan robot grasps

UNDERSTANDING DISPLAY REQUIRMENTS

The first step of this project was meeting with the graduate students working on the Dex-Net project and discussing the display. We came up with a list of information that should be displayed, and briefly discussed the actual implementation - discussing a few different languages and libraries that could be used to develop it. I was also given the dimensions of the screen I would be designing for. 

DIMENSIONS: 1440 x 900 px

REQUIRED ELEMENTS: 

  • Confidence percentage - The robot's percentage of confidence that it will perform a successful grasp
  • Depth camera live video - Displaying exactly what the robot is seeing
  • Picks per hour - Given its current speed, how many objects the robot could potentially pick up each hour
  • Weight of object bin - Show the weight of the object bin, which gets smaller as the robot is more successful 
  • Robot current thought process - Current stage of the algorithm (paused, sensing, planning, moving, error recovery)
  • AUTOLab Logo
  • DEX-NET Logo
  • UC Berkeley Logo

One of the key challenges with this project was creating a simple design for the display that could be understood by non-technical people. There was lot of different data that the lab wanted presented on the display, and finding a way to make it look clean and not simply show different numbers was difficult. 

IDEATION & INITIAL DESIGNS

After discussing the goals of the display at length, I began mocking up some designs. We had come up with a few ideas of how the elements should be presented, which were incorporated into these designs. These included having the confidence percentage be the most significant number, possibly having the object bin weight be shown in a scale bar, and having the robot's thought process in a linear traffic light bar (lighting up the current state).

FEEDBACK:

  • Change yellow and green colors
  • Scale bar is hard to see
  • Show confidence in a scale bar too
